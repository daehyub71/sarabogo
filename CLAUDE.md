## 프로젝트 한 줄 정의

**"살아본 사람에게 묻고, 나에게 맞게 떠난다."**
검증된 후기와 한국관광공사 데이터로, 시니어도 안심하고 한 달을 사는 지역살이 플랫폼.

- **공모전**: 2026 관광데이터 활용 공모전 · 웹·앱 구현 부문
- **필수 요건**: 한국관광공사 OpenAPI를 반드시 활용해 구현·고도화
- **주 타겟**: 은퇴 시니어 장기체류자 (전 국민 대상이되 UX·톤은 시니어 친화)
- **팀 규모**: 2~3인

---

## 핵심 원칙 (개발 판단 기준)

1. **단순함이 방어다.** 스택은 최소로. 화려한 기능보다 "한 축을 깊게".
2. **시니어 우선 UX.** 큰 글씨, 높은 대비, 최소 클릭, 복잡한 가입 없음.
3. **같은 데이터, 다른 질문.** 관광지가 아니라 "여기서 한 달 살 수 있는가"(의료·생활·평지)를 먼저 보여준다.
4. **후기는 긁지 않는다.** 크롤링 금지. 콜드스타트는 **공공누리 제1유형 공공저작물**로 메우고, 본체는 구조화 후기를 자체 축적한다. LLM은 원문에서 **추출**만 하며, 근거 없는 별점은 `null`로 둔다 — 추정 기입은 합성 후기다.
5. **AI는 근거 기반.** LLM 추천은 반드시 우리 DB의 실제 후기를 근거(RAG)로 사용. 환각 방지.

| 기둥 | 내용 | 데이터 출처 |
|------|------|-------------|

| **신뢰** | 구조화된 검증 후기 (항목별 별점) | 공공누리 제1유형 발간물 추출 + 자체 생성 DB |
| **결정** | 후기 반영 맞춤 코스 추천 (근거 후기 노출) | RAG (DB + LLM) |

## 데이터 3층 구조

- **A층 · 지원 프로그램 (입구)**: 지자체 한달살기 공고. **API 없음 → 자체 수집.** MVP는 10~20곳 수기 정제 시드.
- **B층 · 체류·생활 (핵심)**: 관광공사 OpenAPI + 심평원 병원/약국. **공모 필수요건 충족 지점.**
- **C층 · 후기 (신뢰자산)**: 본체는 자체 생성. 콜드스타트는 **공공누리 제1유형 공공저작물 추출**로 해소 (저작권법 제24조의2 · 출처표시만으로 자유이용).

## 기술 스택 (확정)

- **프론트 + API**: Next.js (App Router) / Vercel 배포
- **배포 형태**: **PWA(설치형 웹앱)**. React Native 병행 금지 — 네이티브 앱은 로드맵.
- **DB·인증·스토리지·실시간**: Supabase (PostgreSQL) — `DbPort` 어댑터의 기본 구현
- **배치 수집**: Supabase Edge Functions + Cron (OpenAPI 야간 수집·정제)
- **AI 추천**: GPT / Claude API — `LlmPort` 어댑터, `LLM_PROVIDER`로 교체
- **지도**: 카카오맵 API (시니어 친숙도 우선)
- **UI**: 인스타/페북식 세로 1열 카드 피드 + 시니어 안전장치("더 보기" 버튼·레이블 있는 탭·고대비)

> **설계 결정: Cloud Run 제외.** 2~3인 규모에서 인프라를 하나 더 관리하는 건 순손실.
> 무거운 배치는 Supabase Edge Functions로 대체. "왜 Cloud Run?"이라는 심사 질문을 만들지 않는다.

## MVP 범위 (10월 심사까지)

반드시 구현하는 4개. 이것만 완성해도 수상권.

1. **지역 탐색 지도** — OpenAPI를 시니어 지표로 재편집해 지도+카드 표시
2. **구조화 후기 시스템** — 항목별 별점·태그·서술 + 콜드스타트 시딩
3. **조건 기반 맞춤 추천** — LLM이 후기+관광데이터를 근거로 주차별 코스 생성
4. **카톡 공유** — 결과를 자녀·친구에게 전파

**확장(로드맵으로만, 구현 X)**: 실시간 모집 연동·원스톱 신청 / 총비용 계산기 / 살이族 매칭 / 지자체 인사이트 대시보드

### 검증 (Phase/태스크 완료 조건 — 3단계 모두 통과)

```bash

npm run dev          # 로컬 개발 서버
```

### 반드시 지킬 코드 규칙

- 관광공사 API는 **`src/lib/kto.ts`만 경유**한다. 직접 `fetch` 금지 — `AppName=sarabogo` 누락은 운영계정 승인 거부 사유.
- 심평원은 `src/lib/hira.ts`, 지오코딩은 `src/lib/geocode.ts`를 경유한다.
- 외부 API 키는 서버(Route Handler / Edge Function)에서만 사용한다. 예외는 `KAKAO_JS_KEY`(도메인 제한 필수).
- 화면은 외부 API를 실시간 호출하지 않는다. 배치가 `places`에 저장하고, 화면은 `lib/db`만 읽는다.
- **DB·LLM은 포트+어댑터.** 벤더 SDK(`@supabase/*`, `@anthropic-ai/*`, `openai`)는 `lib/*/adapters/` 안에서만 import한다. 포트 시그니처에 벤더 타입을 노출하지 않는다.
- **인가는 `lib/authz`가 1차 경계**다. RLS는 심층 방어(2차). DB 어댑터를 갈아끼워도 인가 테스트는 통과해야 한다.
- 관리자 콘솔(`app/admin/`)은 **정책 게이트**다. `origin`별로 폼이 갈리며, `curated`에는 **원문 입력 필드를 만들지 않는다.** 모든 쓰기는 `admin_audit`에 남긴다.
- 피드는 **무한 스크롤 금지** — 10개 단위 + "더 보기" 버튼. 하단 탭은 아이콘+한글 레이블. 색·타이포는 `styles/tokens.css`에서만 정의한다.
- LLM 응답의 근거 후기 id는 **DB 실존 여부를 서버에서 검증**한 뒤 렌더링한다.
- 후기 별점은 **nullable**이다. 화면·집계 어디서도 `null`을 `0`으로 치환하지 않는다. 평균은 null 제외 후 계산하고, 화면은 "정보 없음"을 노출한다.
- `origin='public_doc'` 후기는 출처표시 배지(`source_org`·`source_year`·`source_title`·`source_url`) 없이 렌더링하지 않는다 — 공공누리 제1유형의 유일한 의무 조건.

## 절대 하지 말 것 (심사 감점 요인)

- ❌ 관광공사 데이터 크롤링·캐싱 (약관 위반, 운영계정 승인 거부)
- ❌ 구글 리뷰를 DB에 저장 (Places API 정책 위반 — place_id만 저장 가능)
- ❌ 유튜브·블로그·구글맵 스크래핑 (약관 위반 + IP 차단 리스크)
- ❌ 출처 표기 없이 관광공사 이미지 재배포
- ❌ 모든 API 요청에 일관된 AppName 누락 (운영계정 승인 곤란)
- ❌ **LLM이 원문 근거 없이 별점·비용을 추정 기입 (합성 후기)** — "검증된 후기"라는 정체성의 자기부정
- ❌ 공공누리 제2·4유형(상업 금지·변형 금지) 저작물을 시딩에 사용
- ❌ **관리자 콘솔에 블로그·커뮤니티 원문 붙여넣기** — 손으로 하는 크롤링일 뿐이다. 수단이 아니라 행위가 기준
- ❌ AI Hub 등 재배포 금지 데이터셋을 공개 서비스 DB에 적재
