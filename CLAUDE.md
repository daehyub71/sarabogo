> 개발 중 방향이 흔들릴 때 항상 이 문서의 "핵심 원칙"으로 돌아오세요.

## 프로젝트 한 줄 정의

**"살아본 사람에게 묻고, 나에게 맞게 떠난다."**
검증된 후기와 한국관광공사 데이터로, 시니어도 안심하고 한 달을 사는 지역살이 플랫폼.

- **공모전**: 2026 관광데이터 활용 공모전 · 웹·앱 구현 부문
- **필수 요건**: 한국관광공사 OpenAPI를 반드시 활용해 구현·고도화
- **주 타겟**: 은퇴 시니어 장기체류자 (전 국민 대상이되 UX·톤은 시니어 친화)
- **팀 규모**: 2~3인

---

## 핵심 원칙 (개발 판단 기준)

1. **단순함이 방어다.** 스택은 최소로. 화려한 기능보다 "한 축을 깊게".
2. **시니어 우선 UX.** 큰 글씨, 높은 대비, 최소 클릭, 복잡한 가입 없음.
3. **같은 데이터, 다른 질문.** 관광지가 아니라 "여기서 한 달 살 수 있는가"(의료·생활·평지)를 먼저 보여준다.
4. **후기는 긁지 않고 생성한다.** 크롤링 금지. 구조화 후기를 자체 축적한다.
5. **AI는 근거 기반.** LLM 추천은 반드시 우리 DB의 실제 후기를 근거(RAG)로 사용. 환각 방지.

## 서비스 3대 기둥

| 기둥 | 내용 | 데이터 출처 |
|------|------|-------------|
| **탐색** | 시니어 눈으로 재편집된 지역 데이터 (병원·약국·마트·교통·평지) | 관광공사 API + 심평원 API |
| **신뢰** | 구조화된 검증 후기 (항목별 별점) | 자체 생성 DB |
| **결정** | 후기 반영 맞춤 코스 추천 (근거 후기 노출) | RAG (DB + LLM) |

---

## 데이터 3층 구조

- **A층 · 지원 프로그램 (입구)**: 지자체 한달살기 공고. **API 없음 → 자체 수집.** MVP는 10~20곳 수기 정제 시드.
- **B층 · 체류·생활 (핵심)**: 관광공사 OpenAPI + 심평원 병원/약국. **공모 필수요건 충족 지점.**
- **C층 · 후기 (신뢰자산)**: 자체 생성. 콜드스타트는 큐레이션 시딩으로 해소.

---

## 기술 스택 (확정)

- **프론트 + API**: Next.js (App Router) / Vercel 배포
- **DB·인증·스토리지·실시간**: Supabase (PostgreSQL)
- **배치 수집**: Supabase Edge Functions + Cron (OpenAPI 야간 수집·정제)
- **AI 추천**: GPT / Claude API (Next.js API Route에서 호출, RAG 방식)
- **지도**: 카카오맵 API (시니어 친숙도 우선)

> **설계 결정: Cloud Run 제외.** 2~3인 규모에서 인프라를 하나 더 관리하는 건 순손실.
> 무거운 배치는 Supabase Edge Functions로 대체. "왜 Cloud Run?"이라는 심사 질문을 만들지 않는다.

## MVP 범위 (10월 심사까지)

반드시 구현하는 4개. 이것만 완성해도 수상권.

1. **지역 탐색 지도** — OpenAPI를 시니어 지표로 재편집해 지도+카드 표시
2. **구조화 후기 시스템** — 항목별 별점·태그·서술 + 콜드스타트 시딩
3. **조건 기반 맞춤 추천** — LLM이 후기+관광데이터를 근거로 주차별 코스 생성
4. **카톡 공유** — 결과를 자녀·친구에게 전파

**확장(로드맵으로만, 구현 X)**: 실시간 모집 연동·원스톱 신청 / 총비용 계산기 / 살이族 매칭 / 지자체 인사이트 대시보드

## 개발 규칙

워크스페이스 `../CLAUDE.md`의 SDD·TDD·Git 컨벤션·언어 규칙을 따른다. 아래는 이 프로젝트 고유 규칙.

### 검증 (Phase/태스크 완료 조건 — 3단계 모두 통과)

```bash
npm run validate     # lint + tsc --noEmit + test
npm run dev          # 로컬 개발 서버
```

### 반드시 지킬 코드 규칙

- 관광공사 API는 **`src/lib/kto.ts`만 경유**한다. 직접 `fetch` 금지 — `AppName=sarabogo` 누락은 운영계정 승인 거부 사유.
- 심평원은 `src/lib/hira.ts`, 지오코딩은 `src/lib/geocode.ts`를 경유한다.
- 외부 API 키는 서버(Route Handler / Edge Function)에서만 사용한다. 예외는 `KAKAO_JS_KEY`(도메인 제한 필수).
- 화면은 외부 API를 실시간 호출하지 않는다. 배치가 `places`에 저장하고, 화면은 Supabase만 읽는다.
- LLM 응답의 근거 후기 id는 **DB 실존 여부를 서버에서 검증**한 뒤 렌더링한다.

- ❌ 모든 API 요청에 일관된 AppName 누락 (운영계정 승인 곤란)
