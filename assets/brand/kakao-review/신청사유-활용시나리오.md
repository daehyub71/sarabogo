# 카카오맵 신청 사유 및 활용 시나리오

> 카카오 개발자 콘솔 → 추가 기능 신청 → 카카오맵 심사 제출용.
> 아래 "제출용 본문"을 그대로 복사해 붙여넣으면 된다.
> 첨부 서비스 화면: `sarabogo-service-plan.png`

## 제출용 본문 (표준 · 권장)

**[서비스 소개]**

'살아보고(SARABOGO)'는 은퇴 시니어가 지방에서 '한 달 살아보기'를 결정할 때 필요한 정보를 제공하는 웹 서비스입니다. 2026 한국관광공사 관광데이터 활용 공모전(웹·앱 구현 부문) 참가작으로, 한국관광공사 TourAPI와 건강보험심사평가원 공공데이터를 활용해 개발 중입니다.

기존 여행 서비스가 '무엇을 볼 것인가'에 답한다면, 저희는 **'여기서 한 달 살 수 있는가'**에 답합니다. 그래서 관광지 사진보다 병원까지의 거리, 약국 유무, 한 달 실비를 먼저 보여줍니다.

**[신청 사유]**

핵심 기능인 '지역 생활 인프라 지도'를 구현하기 위해 카카오맵이 반드시 필요합니다. 이유는 세 가지입니다.

1. **주소 → 좌표 변환(지오코딩)이 필수입니다.** 심평원이 제공하는 병원·약국 데이터는 좌표가 아닌 '주소' 기반입니다. 지도에 마커를 찍으려면 주소를 좌표로 변환해야 하며, 국내 주소 체계에 가장 정확한 카카오 로컬 API가 필요합니다.

2. **시니어에게 가장 익숙한 지도입니다.** 주 사용자가 60대 이상이며, 이분들이 이미 사용 중인 지도 UI를 그대로 제공하는 것이 학습 부담을 줄이는 가장 확실한 방법입니다. 서비스 전체가 '시니어 친화 UX'(큰 글씨, 최소 클릭)를 원칙으로 설계되었습니다.

3. **'병원까지 차로 몇 분'이 이 서비스의 핵심 지표입니다.** 시니어 장기체류자가 가장 먼저 묻는 질문이며, 숙소 기준 의료시설 접근 시간을 계산해 지역 카드에 노출하는 것이 저희의 핵심 차별점입니다.

**[활용 시나리오]**

- **지역 목록 화면**: 지역별 카드에 '병원 차로 8분 · 한 달 82만원'과 같은 생활 지표를 표시합니다. (현재 구현 완료)

- **지역 상세 지도 화면** (카카오맵 적용 예정)
  - 카카오맵 JavaScript SDK로 해당 지역 지도를 렌더링합니다.
  - 마커를 종류별로 구분해 표시합니다: 병원(적색) · 약국(녹색) · 마트(주황) · 관광지(청색).
  - 병원·약국: 심평원 API로 수집 → 카카오 로컬 API로 주소를 좌표 변환 후 표시.
  - 관광지·숙소: 한국관광공사 TourAPI로 수집 후 표시.
  - 마커 클릭 시 해당 시설의 이름·주소 정보창을 노출합니다.
  - 숙소 기준 가장 가까운 병원까지의 이동 시간을 계산해 표시합니다.

- **맞춤 추천 결과 화면**: 추천된 한 달 코스의 방문지를 지도 위에 함께 표시해 동선을 확인할 수 있게 합니다.

**[사용 예정 API]**

- 카카오맵 JavaScript SDK: 지도 렌더링, 종류별 마커, 정보창
- 카카오 로컬 API: 주소 → 좌표 변환(지오코딩), 생활편의 시설 검색

**[개발 현황]**

Next.js(PWA) + Supabase 기반으로 개발 중이며, 한국관광공사 TourAPI 연동을 완료해 지역 데이터 수집이 동작합니다. 지역 카드 피드 화면은 구현이 완료되어 첨부 이미지에 실제 캡처를 담았습니다. 현재는 로컬 개발 단계이며, 지도 화면 구현을 위해 신청드립니다. 카카오맵 이용 시 지도 위 표시 및 출처 표기 등 카카오 정책을 준수하겠습니다.

## 짧은 버전 (글자수 제한이 있을 경우)

'살아보고'는 은퇴 시니어의 '한 달 살아보기'를 돕는 웹 서비스입니다(2026 한국관광공사 관광데이터 활용 공모전 참가작).

지역별 생활 인프라 지도를 구현하기 위해 카카오맵을 신청합니다. 심평원 병원·약국 데이터가 주소 기반이라 좌표 변환(카카오 로컬 API)이 필수이며, 변환된 좌표를 카카오맵 JavaScript SDK로 렌더링해 병원·약국·마트·관광지를 종류별 마커로 표시할 계획입니다. 숙소 기준 '병원까지 차로 몇 분'을 산출해 보여주는 것이 서비스의 핵심 지표입니다.

주 사용자가 60대 이상이라 가장 익숙한 지도 UI를 제공하는 것이 중요합니다. 현재 Next.js로 개발 중이며 관광공사 TourAPI 연동과 지역 카드 화면 구현을 완료했습니다(첨부 이미지는 실제 화면 캡처). 카카오맵 이용 정책을 준수하겠습니다.

## 작성 시 주의 (반려 방지)

- **서비스 정보 URL**: 카카오는 "실제 서비스 중인 웹 사이트 또는 앱 마켓 URL"을 요구한다.
  "로컬 작업 중"이라고만 쓰면 반려될 수 있다 → **Vercel 배포 후 실 URL을 넣을 것.**
  당장 어렵다면 콘솔 안내에 링크된 '테스트 앱'을 사용한다.
- **첨부 이미지에 개인정보가 없어야 한다** (심사 반려 사유). 제출본은 데모 데이터만 사용했다.
- 지도 화면은 아직 미구현이므로 **"구현 예정(기획안)"임을 명확히 표기**했다.
  실제 카카오맵 캡처인 것처럼 보이게 만들지 않는다.
