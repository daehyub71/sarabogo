## 1. 목표

**"살아본 사람에게 묻고, 나에게 맞게 떠난다."**

검증된 구조화 후기와 한국관광공사 OpenAPI 데이터로, 은퇴 시니어가 안심하고 지역에서 한 달을 살 수 있도록 돕는 웹 플랫폼.

| 항목 | 내용 |
|------|------|
| 공모전 | 2026 관광데이터 활용 공모전 · 웹·앱 구현 부문 |
| 필수 요건 | 한국관광공사 OpenAPI를 반드시 활용해 구현·고도화 |
| 주 타겟 | 은퇴 시니어 장기체류자 (전 국민 대상, UX·톤은 시니어 친화) |
| 팀 규모 | 2~3인 |
| 심사 마감 | 2026년 10월 (1차·최종 심사) |

### 성공 기준 (10월 심사 시점)

1. MVP 4개 기능이 실사용 가능한 수준으로 배포되어 있다 (Vercel 공개 URL).
2. 후기 시딩 50~80건이 출처 표기와 함께 DB에 적재되어, 빈 화면이 없다.
3. AI 추천 결과가 **DB의 실제 후기 id를 근거로 함께 노출**한다 (환각 방지 증명).
4. 관광공사 운영계정이 승인되어 있다.

## 3. 기능 요구사항

### F1 · 지역 탐색 지도

- **FR-1.1** 랜딩에서 큰 버튼 3종(`바다 근처` / `병원 가까운 곳` / `저예산`)으로 로그인 없이 즉시 진입한다.
- **FR-1.2** 지역 카드는 관광지가 아닌 **생활 지표**를 먼저 노출한다: 병원 접근 시간, 평균 한달실비, 후기 수, 평균 별점.
- **FR-1.3** 지역 상세에서 카카오맵 위에 병원·약국·마트·관광지 마커를 종류별로 구분해 표시한다.
- **FR-1.4** 무장애 관광정보(관광공사)를 시니어 편의 지표로 상세에 노출한다.
- **FR-1.5** 관광공사·심평원 데이터는 야간 배치로 `places` 테이블에 정제 저장하며, **화면 요청 시 외부 API를 직접 호출하지 않는다**(응답 지연·쿼터 방어).

### F2 · 구조화 후기

- **FR-2.1** 후기 입력 항목은 아래로 고정한다.

  | 항목 | 타입 | 필수 |
  |------|------|------|
  | 의료접근성 | 별점 1~5 | ✅ |
  | 외로움/커뮤니티 | 별점 1~5 | ✅ |
  | 교통편의 | 별점 1~5 | ✅ |
  | 재방문의사 | 별점 1~5 | ✅ |
  | 한달실비 | 숫자(만원) | ✅ |
  | 자유서술 | 텍스트 | — |
  | 태그 | 문자열 배열 | — |

- **FR-2.2** 시딩 후기(`is_seed = true`)는 **출처(`source`) 없이는 저장할 수 없다** (DB 제약으로 강제).
- **FR-2.3** 지역별 후기 집계(평균 별점, 평균 한달실비)를 배치로 `regions`에 갱신한다.
- **FR-2.4** 후기 작성은 로그인 필요, 조회는 비로그인 허용.

### F3 · 맞춤 추천 (RAG)

- **FR-3.1** 입력: 예산(만원), 건강 배려 필요 여부, 관심사 태그(다중).
- **FR-3.2** 서버는 조건에 맞는 지역의 구조화 후기를 Supabase에서 쿼리해 LLM 프롬프트에 주입한다.
- **FR-3.3** LLM은 **제공된 후기·관광정보 근거 안에서만** 4주차 코스를 생성하고, 각 항목이 참조한 후기 id를 반환한다.
- **FR-3.4** 응답의 근거 후기 id는 화면에 실제 후기 카드로 렌더링된다 (`courses.based_on_review_ids`).
- **FR-3.5** 후기가 부족한 지역은 "에디터 추천 코스"(선탑재)로 대체해 빈 결과를 만들지 않는다.

### F4 · 공유

- **FR-4.1** 추천 결과는 고유 URL을 가지며, 카카오 공유로 전송 가능하다.
- **FR-4.2** 공유 링크는 비로그인 상태에서 열람 가능하다.

## 4. 비기능 요구사항

| 구분 | 요구사항 |
|------|----------|
| **시니어 UX** | 본문 최소 18px, 버튼 터치 영역 최소 48×48px, 명도 대비 WCAG AA(4.5:1) 이상 |
| **가입 마찰** | 탐색·조회는 비로그인 허용. 로그인은 후기 작성 시점에만 요구 |

| **보안** | 모든 외부 API 키는 서버(Route Handler)에서만 사용. 클라이언트 번들 노출 금지. 예외: `KAKAO_JS_KEY` (도메인 제한 필수) |
| **데이터 보호** | `reviews`·`profiles`·사용자 생성 `courses`에 RLS 적용. write는 본인만, read는 공개 |
| **관측** | 관광공사 API 요청 전량에 `AppName=sarabogo` 부착 (운영계정 승인 심사 대상) |

## 5. 기술 스택 (확정)

| 레이어 | 선택 | 근거 |
|--------|------|------|
| 프론트 + API | Next.js (App Router) / Vercel | 프론트·API 단일 배포 |
| DB·인증·스토리지 | Supabase (PostgreSQL) | RLS·Auth 내장, 인프라 관리 최소화 |
| 배치 수집 | Supabase Edge Functions + Cron | 야간 OpenAPI 수집·정제 |
| AI 추천 | Claude API 또는 OpenAI API | Route Handler에서 호출, RAG 방식 |
| 지도 | 카카오맵 API | 시니어 친숙도 우선 |

> **결정: Cloud Run 제외.** 2~3인 팀에서 인프라를 하나 더 관리하는 건 순손실이며,
> "왜 Cloud Run인가?"라는 심사 질문을 자초할 이유가 없다. 무거운 배치는 Edge Functions로 대체한다.

## 6. 데이터 모델

전체 SQL은 `04-database.md`가 원본(single source of truth)이며, 마이그레이션은 그 스키마를 따른다.

| 테이블 | 층 | 설명 |
|--------|----|------|
| `regions` | B | 지역 마스터 + 시니어 집계 지표 (배치 갱신) |
| `programs` | A | 지자체 지원 프로그램 (API 없음 → 자체 수집) |
| `places` | B | 관광지·숙소·병원·약국 정제 캐시 |
| `reviews` | C | 구조화 후기 — **핵심 자산** |
| `courses` | C | 에디터/AI 추천 코스 |
| `profiles` | — | 사용자 프로필 |

**데이터 3층 구조**

- **A층 · 지원 프로그램 (입구)** — 지자체 한달살기 공고. API 없음. MVP는 10~20곳 수기 정제 시드.
- **B층 · 체류·생활 (핵심)** — 관광공사 OpenAPI + 심평원 병원/약국. **공모 필수요건 충족 지점.**
- **C층 · 후기 (신뢰자산)** — 자체 생성. 콜드스타트는 큐레이션 시딩으로 해소.

## 7. 외부 인터페이스

전체 목록과 매핑은 `02-api.md` 참조.

| 그룹 | API | 성격 |
|------|-----|------|
| 관광공사 TourAPI 4.0 | `areaCode2`, `areaBasedList2`, `locationBasedList2`, `searchKeyword2`, `detailCommon2`/`detailIntro2`/`detailInfo2`/`detailImage2`, `searchStay2`, `searchFestival2`, 무장애 관광정보 | **공모 필수** |
| 심평원 | 병원정보서비스, 의료기관별상세정보서비스, 약국정보조회 | 차별화 |
| 카카오 | 맵 JS SDK, 로컬(생활편의 검색), 지오코딩 | 구현 필수 |
| 구글 | Places API | ⚠️ **실시간 조회·표시만. `place_id` 외 저장 금지** |

### 내부 API (Next.js Route Handler)

| 경로 | 역할 |
|------|------|
| `POST /api/recommend` | 조건 수신 → 후기 쿼리 → LLM 호출 → 코스 + 근거 후기 id 반환 |
| `GET /api/tour/*` | 관광공사 프록시 (AppName 자동 주입) |
| `GET /api/medical/*` | 심평원 프록시 |

### 환경변수 (`.env.local`)

```
TOUR_API_KEY=            # 관광공사 (공공데이터포털)
HIRA_API_KEY=            # 심평원 병원/약국
KAKAO_REST_API_KEY=      # 카카오 로컬/지오코딩 (서버)
KAKAO_JS_KEY=            # 카카오맵 JS SDK (클라이언트, 도메인 제한)
GOOGLE_PLACES_KEY=       # 구글 Places (평판 참고용)
ANTHROPIC_API_KEY=       # or OPENAI_API_KEY (LLM 추천)
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
```

## 8. 제약 조건

### 8.1 반드시 지킬 것

- **C-1** 관광공사 API 요청 전량에 고유 `AppName=sarabogo`를 일관 부착한다. 클라이언트 유틸(`lib/kto.ts`)에서 자동 주입하며, 이를 우회하는 직접 호출을 금지한다.
- **C-2** 시딩 후기는 원문 통째 저장이 아니라 **사람이 읽고 구조화 항목에 맞춰 재구성**한 결과만 저장하며, 출처를 명시한다.
- **C-3** 관광공사 이미지 재배포 시 출처를 표기한다 (상세페이지 + 푸터).
- **C-4** LLM 추천은 반드시 DB의 실제 후기를 근거(context)로 사용한다.

### 8.2 절대 하지 말 것 (심사 감점 요인)

- ❌ 관광공사 데이터 크롤링 (약관 위반, 운영계정 승인 거부)
- ❌ 구글 리뷰를 DB에 저장 (Places API 정책 위반 — `place_id`만 저장 가능)
- ❌ 유튜브·블로그·구글맵 스크래핑 (약관 위반 + IP 차단 리스크)
- ❌ 출처 표기 없는 관광공사 이미지 재배포
- ❌ API 키의 클라이언트 노출

### 8.3 일정 제약

관광공사 **운영계정 승인에 약 1주가 소요**되며, 개발 이력(로그)·AppName·실서비스 URL 확인을 거친다.
→ 10월 심사 역산 시 **9월 초에는 신청 가능한 상태**여야 한다.
