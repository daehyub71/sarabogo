# 02 · 활용 API 목록

세 그룹으로 나뉜다: (1) 관광공사 OpenAPI = 공모 필수, (2) 생활·의료 인프라 = 차별화, (3) 지도 = 구현.

## 2) 생활·의료 인프라 API — **차별화 층 (시니어 특화)**

관광공사 API엔 병원·약국이 없음. 여기가 경쟁 서비스가 못 하는 지점. 모두 공공데이터포털에서 신청.

| API | 제공처 | 용도 | 데이터셋 |
|-----|--------|------|----------|
| 병원정보서비스 | 건강보험심사평가원 | 전국 병원 위치·진료과목 | [15001698](https://www.data.go.kr/data/15001698/openapi.do) |
| 의료기관별상세정보서비스 | 심평원 | 시설·진료과목·교통·장비·특수진료. "병원 차로 8분" 근거 | [15001699](https://www.data.go.kr/data/15001699/openapi.do) |
| 약국정보서비스 | 심평원 | 약국 위치 | [15001673](https://www.data.go.kr/data/15001673/openapi.do) |

### 검증된 엔드포인트 (2026-07-12 실호출 확인)

기관코드는 **B551182**. 아래 두 경로는 키 없이 호출 시 `401`이 떠서 **경로가 정상**임을 확인했다.

```
병원: https://apis.data.go.kr/B551182/hospInfoServicev2/getHospBasisList
약국: https://apis.data.go.kr/B551182/pharmacyInfoService/getParmacyBasisList
```

> ❌ `hospInfoServicev3`, `MadmDtlInfoService2/getDtlInfo2`는 `500`을 반환한다 — **존재하지 않는 경로다.**
> 의료기관별상세정보의 정확한 오퍼레이션명은 구현 시 마이페이지 → 개발계정 상세보기의 **End Point** 란에서 확인할 것.

**응답 포맷은 XML이 기본**이다(관광공사와 달리 `_type=json`이 안 먹을 수 있음). `lib/hira.ts` 작성 시 실호출로 먼저 확인한다.

### 진단법 — 403 vs 401 (관광공사에서 배운 것)

| 응답 | 의미 |
|------|------|
| 키 없이 `401` | 엔드포인트 경로가 **정상** |
| 키 없이 `500` | 경로가 **틀림** |
| 키 넣고 `403` | 경로는 맞으나 **그 서비스에 키가 아직 활성화 안 됨** (활용신청 승인/전파 대기) |
| 키 넣고 `200` + `resultCode` | 정상. 단 `resultCode`를 반드시 검사할 것 |

> **교훈: 반드시 실호출로 검증한다.** 관광공사에서 문서 규칙(`AppName`)이 실제와 달라 모든 호출이 실패할 뻔했다.
> 심평원·카카오도 mock이 아니라 실 API로 확인한 뒤 코드에 박는다.

## 3) 지도·지오코딩 API — **구현 필수**

| API | 용도 | 비고 |
|-----|------|------|
| 카카오맵 API | 지도 렌더링·마커·길찾기 | 시니어 친숙도 우선 |
| 카카오/네이버 지오코딩 | 주소→좌표 변환 | 심평원 병원 데이터가 주소 기반이라 필수 |
| 카카오 로컬 API | 마트·편의점 등 생활편의 검색 | 생활 인프라 보완 |
| 구글 Places API | 장소 평판 실시간 참고 | **⚠️ 리뷰 저장 금지. place_id만 저장 가능. 실시간 조회·표시만** |

## 신청·운영 주의사항

### 개발계정 → 운영계정 승인
- 관광공사는 개발계정으로 개발 후, 완료 시점에 운영계정 신청 → 개발 이력(로그)·앱이름(MobileApp)·실서비스 URL 확인해 승인(약 1주 소요).
- **모든 API 요청에 고유 앱 식별자 `MobileApp=sarabogo`를 일관되게 넣을 것.** 초기부터 습관화.
- ⚠️ **`AppName` 파라미터는 TourAPI에 존재하지 않는다.** 보내면 `resultCode 10`으로 요청이 실패한다 (2026-07-12 실호출 확인).

### 저작권·출처
- TourAPI 데이터는 상업적 활용 가능한 공공저작물이나 출처 명시 필수.
- **이미지는 재배포 시 출처 표기 필수.** 서비스 하단·상세페이지에 출처 문구 삽입.

### 환경변수 (`.env.local`)
```
TOUR_API_KEY=          # 관광공사 (공공데이터포털)
HIRA_API_KEY=          # 심평원 병원/약국
KAKAO_REST_API_KEY=    # 카카오맵/로컬/지오코딩
KAKAO_JS_KEY=          # 카카오맵 JS SDK (클라이언트)
GOOGLE_PLACES_KEY=     # 구글 Places (평판 참고용)
OPENAI_API_KEY=        # or ANTHROPIC_API_KEY (LLM 추천)
NEXT_PUBLIC_SUPABASE_URL=
SUPABASE_SERVICE_ROLE_KEY=
```
> 키는 절대 클라이언트에 노출 금지. 서버(API Route)에서 프록시.
